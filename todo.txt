# 네이버 부동산 429 오류 (Too Many Requests) 대응 작업 내용

## 1. 완료된 작업 (Completed)
- `app/config.py`: 크롤링 딜레이를 기본 10초에서 15초로 상향하고 최대 재시도 횟수를 3회에서 5회로 증가
- `crawler/naver_crawler.py`: `_reset_session` 메소드를 도입하여 세션 및 쿠키를 초기화하도록 수정
- `crawler/naver_crawler.py`: 지수 백오프(`_wait_with_backoff`) 로직 구현하여 서버 차단 시 2배수 대기 시간 적용
- `crawler/naver_crawler.py`: PC 버전 API(`new.land.naver.com`)가 보안으로 막히는 문제를 해결하기 위해, 모바일 웹 및 모바일 클러스터 API(`m.land.naver.com`)로 전면 개편
  - `_get_region_code`: 모바일 웹페이지 HTML 스크래핑 방식으로 전환하여 `cortarNo` 추출
  - `_search_properties`: 모바일 클러스터 목록 API(`clusterList`)와 매물 목록 API(`articleList`)를 사용하여 검색 로직 수정
  - `_search_properties`: 각 클러스터별로 페이징 처리 지원 (과도한 부하 방지를 위해 클러스터/페이지 제한 적용)
  - `_parse_property`: 모바일 API JSON 구조(`atclNo`, `atclNm`, `prc` 등)에 맞게 파싱 로직 업데이트
  - `_get_mobile_headers`: 모바일 전용 헤더 (User-Agent 등) 생성 함수 분리 적용

## 2. 향후 과제 / 미완료 작업 (Pending/Future Work)
- 모바일 클러스터 API는 상세 주소(`address`)를 기본적으로 제공하지 않으므로, 필요한 경우 추가 API 호출이나 단지 정보 매핑을 통해 상세 주소를 보완하는 작업
- 클러스터 데이터 수집 시 클러스터별 최대 5페이지, 클러스터 10개 제한을 하드코딩해 두었으므로, 이를 `app/config.py`로 분리하여 유연하게 조절할 수 있도록 개선
- 현재는 모바일 API만으로 거의 대응이 가능하지만, 모바일 API마저 차단될 경우를 대비해 Selenium 백업 로직을 최신 모바일 구조에 맞게 고도화
