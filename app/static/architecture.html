시스템 아키텍처

개요

이 시스템은 네이버 부동산과 국토교통부 공공 API에서 부동산 데이터를 수집하여 지도 기반 인터페이스로 제공하는 웹 애플리케이션입니다. 주요 진입점은 app/main.py의 FastAPI 앱이며, 사용자 요청은 app/routers/api.py를 통해 처리됩니다.

데이터 흐름도

개요

사용자의 검색 요청부터 결과 표시까지의 데이터 흐름을 보여줍니다. 프론트엔드의 searchProperties() 함수가 API를 호출하고, 백엔드의 search_properties() 함수가 크롤링 서비스를 통해 데이터를 수집합니다.

크롤링 프로세스 흐름

개요

크롤러가 외부 API에서 데이터를 수집하는 과정을 보여줍니다. search_region() 메서드가 시작점이며, _get_region_code()와 _search_properties()를 거쳐 최종 결과를 반환합니다. 429 오류 발생 시 _rotate_user_agent()와 _random_delay()를 통해 재시도합니다.

개요

사용자가 애플리케이션을 사용하는 과정을 보여줍니다. SearchApp 클래스가 애플리케이션의 핵심 로직을 관리하며, init() 메서드로 시작하여 searchProperties()로 검색을 실행합니다.

기술 스택

Backend (app/)

  • • main.py - FastAPI 앱 진입점
  • • routers/api.py - REST API 엔드포인트
  • • services/crawler_service.py - 비즈니스 로직
  • • config.py - 환경 설정

Crawler (crawler/)

  • • naver_crawler.py - 네이버 부동산 크롤러
  • • molit_crawler.py - 국토교통부 API 크롤러
  • • search_region() - 지역 검색 진입점
  • • _parse_property() - 데이터 파싱

Frontend (app/static/)

  • • search.js - 검색 페이지 로직
  • • initMap() - Leaflet 지도 초기화
  • • drawCharts() - Chart.js 차트 생성
  • • searchProperties() - API 호출

Templates (app/templates/)

  • • index.html - 홈페이지
  • • search.html - 검색 페이지
  • • guide.html - 사용 설명서
  • • architecture.html - 이 문서
