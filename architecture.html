시스템 아키텍처

개요

이 시스템은 네이버 부동산과 국토교통부 공공 API에서 부동산 데이터를 수집하여 지도 기반 인터페이스로 제공하는 웹 애플리케이션입니다.

데이터 흐름도

개요

사용자의 검색 요청부터 결과 표시까지의 데이터 흐름을 보여줍니다.

sequenceDiagram participant User as 사용자 participant FE as 프론트엔드 participant API as FastAPI 서버 participant CS as 크롤링 서비스 participant Cache as 캐시 시스템 participant Ext as 외부 API User->>FE: 지역 선택 & 검색 요청 FE->>API: GET /api/v1/search?locations=... API->>CS: search_properties() loop 각 지역별 CS->>Cache: 캐시 확인 alt 캐시 존재 & 유효 Cache-->>CS: 캐시된 데이터 반환 else 캐시 없음 or 만료 CS->>Ext: 크롤링 요청 alt 네이버 부동산 성공 Ext-->>CS: 매물 데이터 else 실패 (429 오류) CS->>CS: 더미 데이터 생성 end CS->>Cache: 새 데이터 캐시 저장 end end CS-->>API: 매물 목록 반환 API-->>FE: JSON 응답 FE->>User: 지도 및 목록 표시 FE->>User: 차트 시각화
